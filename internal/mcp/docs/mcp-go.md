mark3labs/mcp-go 不是一个包,是一个模块,底下分三块,先搞清楚你要干哪件事:

```
┌────────┬─────────────────────────────────────────┬────────────────────────┐
│  子包  │                 干什么                   │      你会不会用到       │
├────────┼─────────────────────────────────────────┼────────────────────────┤
│ mcp    │ 纯协议数据结构(最大最杂)                  │ 很少直接翻,当字典用      │
├────────┼─────────────────────────────────────────┼────────────────────────┤
│ server │ 写一个 MCP 服务端(暴露工具给 AI 用)       │ 想做 server 就只看这个   │
├────────┼─────────────────────────────────────────┼────────────────────────┤
│ client │ 写一个 MCP 客户端(连别人的 server)        │ 想做 client 就只看这个  │
└────────┴─────────────────────────────────────────┴────────────────────────┘
```

如果你的目标是「连别的 MCP server」,你压根不需要通读 mcp 包那几百个类型——你只看 client 包就行,而 client 包导出的东西少得多。

mcp 包大,是因为 server 和 client 都依赖它当底层数据定义。它是地基,不是你直接住的房间。
