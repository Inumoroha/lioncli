# 联网操作指引（web-access）

你可以通过两个本地工具联网。它们各管一段，配合使用：

- `web_search(query)`：拿关键词去搜索引擎，返回若干条 {title, link, snippet}。
  用来**发现**信息源，不返回页面正文。
- `web_fetch(url)`：抓取**单个** URL 的正文（已做正文提取/截断）。
  用来**读取**某个具体页面的内容。

## 标准流程

1. **先搜后抓**：用户问的是开放性/实时性问题时，先用 `web_search` 把问题拆成
   1~3 个精准 query 去搜，从返回的 link 里挑最相关的 2~3 个。
2. **再逐个抓取**：对挑中的 link 调 `web_fetch` 读正文。不要凭 snippet 直接下结论——
   snippet 往往残缺或过期。
3. **交叉印证**：涉及数字、日期、版本号等硬事实时，至少抓两个独立来源比对一致再回答。
4. **给出处**：回答末尾用 markdown 链接列出你实际读过的 URL，方便用户复核。

## query 怎么写

- 用名词短语,去掉口语化的疑问词("怎么"、"是不是")。
- 含实体时带上专有名词原文(英文名/型号/版本),命中率更高。
- 一次搜不到就换角度重搜,而不是反复抓同一个差链接。

## 注意

- 已知确切 URL 时,跳过搜索直接 `web_fetch`。
- 抓取失败(404/超时)不要硬重试同一个 URL,回到搜索换一个来源。
- 不联网就能答的常识性问题,别为了联网而联网。
